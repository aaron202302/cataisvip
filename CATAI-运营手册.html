2. 首次部署指南

第一步：创建 GitHub 仓库

  1. 访问 https://github.com/new
  2. 填写信息：
    • Repository name：catai
    • Description：AI Ecosystem Portal
    • Public（公开）
    • 不要勾选 "Add a README file"
  3. 点击 Create repository

第二步：推送代码到 GitHub

# 添加远程仓库（替换为你的用户名）
git remote add origin https://github.com/你的用户名/catai.git

# 推送代码
git branch -M main
git push -u origin main

# 推送版本标签
git push origin v1.0

第三步：部署到 Vercel

  1. 访问 https://vercel.com/signup
  2. 使用 GitHub 账号 登录
  3. 点击 Add New Project
  4. 选择 catai 仓库
  5. 配置保持不变，点击 Deploy
  6. 等待 1-2 分钟，部署完成

第四步：配置域名

4.1 在 Vercel 添加域名

  1. 进入项目 → Settings → Domains
  2. 输入 catai.cc.cd
  3. 点击 Add

4.2 在 Cloudflare 配置 DNS

  1. 登录 Cloudflare，选择域名
  2. 进入 DNS → Records
  3. 添加 CNAME 记录：
    • Type：CNAME
    • Name：@
    • Target：cname.vercel-dns.com
    • Proxy status：DNS only（关闭代理）
  4. 保存，等待 5-10 分钟生效

第五步：提交 Google 收录

  1. 访问 Google Search Console
  2. 添加资源：https://catai.cc.cd
  3. 验证域名（DNS 验证）
  4. 提交 Sitemap：https://catai.cc.cd/sitemap.xml

7. 故障排查

网站无法访问

检查步骤：
  1. 检查 Vercel 状态：https://vercel.com/dashboard
  2. 检查域名解析：ping catai.cc.cd
  3. 检查部署日志：Vercel Dashboard → 项目 → Deployments

页面显示异常

常见原因：

  • JSON 格式错误：访问 https://jsonlint.com/ 验证
  • 构建失败：本地运行 npm run build 测试
  • 缓存问题：清除浏览器缓存，硬刷新 Ctrl + F5

Logo 不显示

解决方案：

  1. 手动上传 Logo 到 public/logos/ 文件夹
  2. 修改 tools.json 中的 logo 路径："logo": "/logos/工具名.png"

8. 备份与回滚

自动备份（GitHub）

所有代码自动保存在 GitHub，无需手动备份。

回滚到上一版本

# 查看最近提交
git log --oneline -5

# 回滚到上一版本
git reset --hard HEAD~1
git push --force

回滚到指定版本

# 查看所有版本
git log --oneline

# 回滚到 v1.0
git reset --hard v1.0
git push --force

只回滚特定文件

# 恢复文件到指定版本
git checkout v1.0 -- src/data/tools.json
git commit -m "回滚 tools.json 到 v1.0"
git push
