1. 项目概述与定位

CATAI（读作 /kəˈtaɪ/）是一个面向全球用户的 AI 生态门户（AI Ecosystem Portal）。核心目标是成为用户发现和了解 AI 工具、模型、资源的一站式入口。

6. 国际化（多语言）系统

支持的语言

语言代码语言翻译文件内容回退
enEnglishmessages/en.json默认语言，无需回退
zh中文messages/zh.json工具名称有 nameZh，描述有 description
ja日本語messages/ja.jsonUI 翻译完整，工具/博客内容回退英文
esEspañolmessages/es.jsonUI 翻译完整，工具/博客内容回退英文
frFrançaismessages/fr.jsonUI 翻译完整，工具/博客内容回退英文

关键文件

  • src/i18n/routing.ts — 定义支持的语言列表和默认语言
  • src/i18n/request.ts — 根据 URL 加载对应翻译文件
  • src/i18n/navigation.ts — 提供带语言切换的 Link、redirect 等
  • src/middleware.ts — 拦截请求，自动识别语言并跳转

新增翻译的步骤

  1. 在 messages/en.json 中添加英文键值对
  2. 在 messages/zh.json 中添加对应中文翻译
  3. 同步更新 ja.json、es.json、fr.json
  4. 在组件中使用 useTranslations('namespace') 引用
⚠️ 重要：修改任何语言文件后，必须同步更新其他 4 个语言文件。否则缺失的 key 会显示 key 名称而非翻译内容。

7. 数据文件说明

7.1 tools.json（工具基础数据）

存放 37 款 AI 工具的基础信息，每款工具结构：

{
  "slug": "chatgpt",           // URL 标识（唯一）
  "name": "ChatGPT",           // 英文名
  "nameZh": "ChatGPT",         // 中文名（可留空，用英文名）
  "description": "...",         // 中文描述
  "descriptionEn": "...",       // 英文描述
  "url": "https://chat.openai.com",  // 官网地址
  "logo": "https://www.google.com/s2/favicons?domain=chat.openai.com&sz=128",
  "category": "Chat",          // 分类
  "tags": ["Freemium", "API"], // 标签
  "featured": true             // 是否推荐（首页置顶）
}

7.2 toolDetails.json（工具详情）

以 slug 为 key，存放每款工具的详细信息：

    "pros": { "en": [...], "zh": [...] },
    "cons": { "en": [...], "zh": [...] },

}

常用命令

添加新工具的步骤

  1. 在 src/data/tools.json 中添加基础信息（参考现有格式）
  2. 在 src/data/toolDetails.json 中添加详情（功能/定价/优缺点）
  3. 运行 npm run build 验证无错误
  4. git push 到 GitHub，Vercel 自动部署
  5. 也可使用 tool-editor.html 可视化编辑后导出 JSON

添加新博客的步骤

  1. 打开 blog-editor.html（浏览器直接打开）
  2. 填写标题、摘要、正文（支持 Markdown）
  3. 导出 JSON → 粘贴到 src/data/blogPosts.json
  4. 运行 npm run build 验证
  5. git push 部署
⚠️ GitHub 推送注意：国内网络可能连不上 GitHub，需要先运行 git config http.postBuffer 524288000 增大缓冲区。如果仍然失败，等几分钟后重试。

15. 常见问题 FAQ

Q: 为什么推送 GitHub 经常失败？

A: 国内网络访问 GitHub 不稳定。解决方法：先运行 git config http.postBuffer 524288000，然后重试。如果还不行，等几分钟再试。

Q: 修改了翻译文件但没效果？

A: 需要同时修改 5 个语言文件（en/zh/ja/es/fr.json），否则缺失的 key 会显示原始 key 名。修改后需要 git push 触发 Vercel 重新部署。

Q: 新增工具后为什么 Logo 不显示？

A: Logo 使用 Google Favicon Service（www.google.com/s2/favicons），如果目标网站没有 favicon 就会显示空白。可以手动指定其他 Logo URL。

Q: 本地构建成功但线上报错？

A: 检查 Vercel 的部署日志。常见原因：JSON 文件格式错误、缺少翻译 key、图片 URL 不可访问。

Q: 博客发布一定要 push 代码吗？

A: 目前是的。后续计划接入 Sanity CMS，实现可视化发布，无需碰代码。

Q: 工具详情页的 "Latest Update" 日期怎么写？

A: 在 toolDetails.json 的 latestUpdate 字段中填写最新动态，建议更新到当月（如 "July 2026"）。

Q: 为什么 ja/es/fr 语言的工具名称和描述还是英文？

A: 工具数据（tools.json）只有中英文翻译，其他语言回退到英文显示。如果未来需要日语/西语/法语的工具描述，需要在 tools.json 中增加对应字段。
